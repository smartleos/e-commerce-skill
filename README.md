# e-commerce-skills｜電商詳情頁 Skill

這是一個用於沉澱電商實戰工作流的 skill 包。當前已包含 2 個詳情頁相關 skill，後續可以繼續按同樣結構追加新的電商 skill。

當前可用能力：

1. `ecommerce-content-wireframe`：從產品資訊出發，梳理詳情頁內容結構、詳情頁文案，並生成可執行的手機長圖 HTML 黑白線框稿。
2. `ecommerce-detail-page-visual-design`：在已有詳情頁原型圖、包裝圖及其他可選資料的基礎上，逐屏判斷資訊任務和視覺表達，推薦統一視覺風格，一屏一屏生成並確認，最後拼接成 750px 詳情頁長圖。

## Skill 1：ecommerce-content-wireframe

電商詳情頁內容黑白線框稿。

### 安裝

在 Claude Code、Codex、WorkBuddy 等支援 skills 的工具中執行：

```text
/install-skill saibodafu/e-commerce-skill/ecommerce-content-wireframe
```

### 使用場景

當你想梳理一個商品的電商詳情頁內容時使用。skill 會先要求你補充產品名、產品定位、目標人群、核心場景和核心賣點，再一步步生成詳情頁內容。

### 工作流程

第一階段：先收集精準資訊

使用者先填寫完整產品資訊表。如果暫時不確定某一項，可以寫“待定”，但產品名、目標人群、核心場景和核心賣點建議儘量具體。

第二階段：再進入共創

產品資訊確認後，skill 按固定順序逐層推進：

1. 核心價值承諾層
2. 痛點錨定層
3. 賣點強化層
4. 使用場景層
5. 產品身份卡
6. 決策收口層

最終會先整理成完整的詳情頁內容文件，再生成可直接執行的 HTML 黑白線框稿。

### 示例輸入

```text
使用 ecommerce-content-wireframe，幫我梳理一款兒童電解質水的詳情頁內容。

產品名：小K水
產品定位：兒童日常補水與電解質補充飲品
目標人群：3-12歲兒童家長
核心賣點：低糖、好喝、補充電解質、兒童友善
```

### 也可以這樣說

```text
幫我梳理一個新品的電商詳情頁內容。
```

```text
用 ecommerce-content-wireframe，幫我基於完整產品資訊生成詳情頁黑白線框稿。
```

```text
我有一個貓糧產品，幫我先整理詳情頁內容結構。
```

### 輸出特點

- 先收集精準產品資訊，再生成詳情頁內容
- 每一步都給多個選擇，並指引使用者選擇、融合或補充資訊
- 關鍵欄位不足時先補齊，非關鍵細節不足時再做輕量假設並明確標註
- 不編造平台規則、銷量、轉換率、認證、競品資料或客戶案例
- 輸出可以交給營運、內容、設計或商品團隊繼續執行

## Skill 2：ecommerce-detail-page-visual-design

電商詳情頁逐屏視覺設計與長圖拼接。

### 安裝

在 Claude Code、Codex、WorkBuddy 等支援 skills 的工具中執行：

```text
/install-skill saibodafu/e-commerce-skill/ecommerce-detail-page-visual-design
```

推薦在 Codex 下使用，並配合 ChatGPT Image 2 模型執行；這個組合對電商詳情頁視覺生成和中文文字呈現效果最好，更適合需要完整中文標題、賣點和長圖文案的場景。

### 使用場景

當你已經有詳情頁原型圖、包裝圖，並希望進入視覺設計階段時使用。詳情頁文案、HTML 圖、產品圖、品牌規範、目標平台等資料不限，有則使用，沒有則按現有資訊推進並標註假設。

### 工作流程

skill 會按固定順序推進：

1. 逐屏分析每一屏的資訊任務
2. 確認每一屏的資訊任務、視覺表達類型、核心視覺元素和風險點
3. 根據產品資訊、品牌包裝圖和詳情頁文案，推薦 3 個適合電商詳情頁的視覺風格方向
4. 基於原型圖中的每一屏文案、版式和資訊層級，逐屏生成 AI 生圖提示詞，只生成提示詞，不直接生成圖片
5. 使用 ChatGPT Image 2 按順序一屏一屏生成詳情頁設計
6. 每一屏修改和確認後再進入下一屏
7. 所有單屏確認後縱向拼接成 750px 長圖
8. 檢查漏屏、錯序、重複、文字裁切和產品裁切
9. 輸出最終長圖和品檢結果

### 示例輸入

```text
使用 ecommerce-detail-page-visual-design，我已經有詳情頁原型圖和包裝圖，幫我逐屏分析視覺表達，再按順序一屏一屏生成，最後拼接成 750px 長圖。
```

### 也可以這樣說

```text
我有詳情頁 HTML 圖和包裝圖，幫我判斷每一屏適合怎麼設計。
```

```text
幫我把確認好的單屏詳情頁按順序拼成長圖，寬度 750px。
```

### 輸出特點

- 視覺設計流程中預設先分析、再定風格、再逐屏生成和確認
- 長圖拼接後必須檢查漏屏、錯序、重複、文字裁切和產品裁切
- 原型圖和包裝圖是啟動必需材料，其他材料不限，有則使用
