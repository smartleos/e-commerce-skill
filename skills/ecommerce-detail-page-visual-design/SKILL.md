# 電商詳情頁逐屏視覺設計與長圖拼接

## 使用場景

當使用者已經完成詳情頁文字資訊梳理，並提供詳情頁 HTML 圖、原型圖、產品圖、包裝圖、品牌圖或單屏設計稿時，使用本 skill 協助完成後續視覺設計流程。

本 skill 不負責從零梳理詳情頁內容結構；如果使用者還沒有詳情頁文案、模組順序或產品資訊，應先使用 `ecommerce-content-wireframe`。

## 觸發條件

使用者需求涉及以下任一情況時觸發：

- 已有詳情頁文案，希望逐屏判斷每一屏怎麼設計
- 已有詳情頁 HTML 圖、原型圖或線框稿，希望轉成視覺設計
- 希望推薦 3 個適合詳情頁的統一視覺風格方向
- 希望一屏一屏生成電商詳情頁設計，並逐屏確認修改
- 希望把確認後的單屏詳情頁縱向拼接成 750px 長圖
- 希望檢查長圖是否漏屏、錯序、重複、文字裁切或產品裁切

## 核心原則

- 保留使用者原始流程，不把它改寫成複雜方案。
- 先分析，不直接出圖。
- 先統一視覺風格，再逐屏作圖。
- 每次只處理一屏，等使用者確認後再進入下一屏。
- 修改時只修改當前屏，保留使用者已經確認的部分。
- 凡涉及生成詳情頁單屏圖片或長圖視覺稿，必須呼叫 `ChatGPT Image 2`；不要改用其他生圖模型替代。
- 不擅自改寫已確認文案；但若發現內部溝通文字、非銷售內容或不合格文案，必須先攔截，提出替換方案並等待使用者確認後才能生圖。
- 成品畫面只能出現已確認的「可上畫面文案」。資訊任務、分析結論、設計指示、選擇理由、風險提醒、假設、自檢結果和對話話術一律不得進入圖片。
- 詳情頁必須有分段 CTA。10 屏預設檢查第 1、4、7、10 屏；其他屏數檢查首屏、中段節點與末屏。
- 不編造功效、資料、認證、平台規則、廣告法結論或競品資訊。
- 如涉及最新平台規則、合規要求、榜單、價格、店鋪或投放機制，先連網核驗並標註來源和時間。

## 必要輸入

開始前，優先確認使用者是否已經提供原型圖和包裝圖。其他材料不限，有則使用，沒有則基於現有資訊推進，並在輸出中標註“假設”。

```text
【原型圖】：必須提供，用於判斷每一屏結構、文案位置和資訊順序
【包裝圖】：必須提供，用於提取品牌色、產品氣質、包裝元素和視覺統一方向
【其他可選材料】：詳情頁文案、HTML 圖、產品圖、品牌規範、目標平台、字型要求、禁用表達、必須露出的資質或賣點、已標註的「可上畫面文案」
```

如果缺少原型圖或包裝圖，先請使用者補充。其他材料不作為啟動門檻；缺少時做輕量合理假設，並在輸出中標註“假設”。

如果輸入材料沒有區分「可上畫面文案」與內部策劃內容，先由本 skill 逐屏辨識並列出文字白名單，經使用者確認後再進入生圖提示詞。

## 觸發後的開始語

當使用者想進入詳情頁視覺設計流程時，使用下面的話術，不要直接套用“假如你是……”：

```text
我會基於你已經整理好的詳情頁文案、原型圖和產品圖，按“逐屏文案與 CTA 審核 - 視覺分析確認 - 3 個視覺風格方向 - 逐屏生圖提示詞 - 單屏生成 - 修改確認 - 長圖拼接 - 最終品檢”的順序推進。

我先不直接出圖，會先區分可上畫面文案與內部文字，檢查 CTA 分布和文案銷售力，再判斷每一屏的資訊任務、適合的視覺表達、核心視覺元素和需要避開的風險點。確認後，再進入統一視覺風格和逐屏作圖。

請先提供詳情頁文案/HTML 圖、原型圖、產品圖或品牌包裝圖。已經提供的話，我會先從逐屏分析開始。
```

如果使用者已經提供完整材料，可以直接進入“階段 1：逐屏分析”。

## 工作流程

按以下順序推進，不要跳步：

1. 逐屏分析
2. 逐屏分析確認
3. 推薦 3 個視覺風格方向
4. 逐屏生成 AI 生圖提示詞
5. 逐屏生成設計
6. 當前屏修改確認
7. 進入下一屏
8. 長圖拼接
9. 最終品檢

## 階段 1：逐屏分析

目標：先完成文案與 CTA 前置審核，再根據每一屏可上畫面文案判斷轉換任務和視覺表達方向。

輸出表格：

| 屏序 | 可上畫面核心文案 | 文案可用性 | CTA | 資訊任務 | 視覺表達類型 | 核心視覺元素 | 禁止上畫面內容/風險點 |
|---|---|---|---|---|---|---|---|

判斷標準：

- 「可上畫面核心文案」只能填寫消費者實際會看到的主標題、副標題、輔助文案、CTA 和必要告知。
- 「文案可用性」標記為通過或攔截。不得因文字已經出現在原型圖中，就預設它可以進入成品。
- 內部分析、資訊任務、設計建議、提示詞說明、選擇理由、風險提醒、假設、自檢結果、版本備註、對話話術都列入「禁止上畫面內容」。
- 除成分、規格、用法與必要告知外，文案應至少具備痛點、癢點、爽點、具體情境、共感、動作、結果感、購買利益或行動推進中的一項。
- 平鋪直述、空泛、無情緒、無動作、無法同理、沒有共感或與當前屏銷售任務無關的文案，標記為攔截。
- 10 屏詳情頁預設在第 1、4、7、10 屏檢查 CTA；4 屏以上的非 10 屏頁面按首屏、每隔約 3 屏與末屏檢查；1 至 3 屏可合併首屏與末屏 CTA，不強制另設中段 CTA。CTA 缺失、過密或與內容無關時標記為攔截。
- 資訊任務要具體，例如“首屏建立品類認知”“解釋核心賣點”“證明成分/材質”“製造使用場景代入”“降低購買顧慮”“決策收口”。
- 視覺表達類型要可執行，例如“產品主視覺大圖”“場景化生活方式圖”“成分/結構拆解圖”“對比圖”“步驟圖”“證據背書圖”“參數資訊圖”。
- 核心視覺元素要能指導畫面製作，例如產品、包裝、手部、場景、符號、圖示、質感、局部特寫、對比區域。
- 風險點要指出可能涉及廣告法、功效誇大、絕對化用語、醫療化表達、虛假承諾或無法證明的資料表達。

如有任一屏被攔截，在表格後輸出：

```text
生成前攔截清單

第 X 屏
原文：
攔截原因：
建議替換文案：
替換依據：痛點/癢點/爽點/情境/共感/動作/利益/CTA 承接
```

攔截清單中的建議文字只是候選方案，不得直接視為已確認文案。使用者確認或修改前，不進入視覺風格推薦、生圖提示詞或圖片生成。

階段收尾話術：

```text
以上是逐屏文案、CTA 與視覺分析。你可以確認這版分析，也可以指定某一屏調整可上畫面文案、CTA、資訊任務、視覺類型或風險判斷。所有攔截項確認後，我才會進入 3 個視覺風格方向推薦。
```

## 階段 2：逐屏分析確認

目標：讓使用者先確認逐屏分析是否準確，再進入視覺風格方向推薦。

如果使用者沒有明確確認，不要直接進入生圖。可以繼續修改某一屏的可上畫面文案、CTA、資訊任務、視覺表達類型、核心視覺元素或風險判斷。

只有全部屏的文案可用性均為「通過」、CTA 節點確認完成，才能進入下一階段。

確認話術：

```text
如果這版逐屏分析沒有問題，你可以回覆“確認”，我會基於已提供的產品資訊、品牌包裝圖和詳情頁文案，推薦 3 個適合電商詳情頁的視覺風格方向。
```

## 階段 3：推薦 3 個視覺風格方向

目標：根據已提供的產品資訊、品牌包裝圖和詳情頁文案，推薦 3 個適合電商詳情頁的視覺風格方向。每個方向用一句話描述風格、構圖、字型方向，方便複用在每一屏作圖前的統一視覺風格提示詞。

輸出格式：

```text
視覺風格方向 1：
一句話描述：[風格、構圖、字型、色彩質感、適合複用的螢幕類型]

視覺風格方向 2：
一句話描述：[風格、構圖、字型、色彩質感、適合複用的螢幕類型]

視覺風格方向 3：
一句話描述：[風格、構圖、字型、色彩質感、適合複用的螢幕類型]

推薦優先方向：
[說明為什麼這個方向最適合當前產品、品牌包裝和轉換目標]
```

要求：

- 優先延展品牌包裝已有顏色、材質和字型氣質。
- 不要只寫“高階感、年輕化、簡約風”等空泛詞。
- 風格描述要能直接複用到作圖提示詞裡。
- 如果使用者選擇其中一個方向，後續每一屏都沿用該方向。

階段收尾話術：

```text
你可以直接選一個方向，例如“選 2”；也可以說“1 和 3 融合”，或告訴我想更偏高階、清爽、專業、場景化、貨架感。確認風格後，我會基於原型圖逐屏生成可用於 AI 生圖的提示詞。
```

## 階段 4：逐屏生成 AI 生圖提示詞

目標：基於使用者提供的詳情頁原型圖，按照原型圖中的每一屏文案、版式和資訊層級，逐屏生成可用於 AI 生圖的提示詞。

這一階段只生成提示詞，不直接生成圖片。

每一屏提示詞必須包含：

```text
第 X 屏 AI 生圖提示詞
主體：[畫面主體]
場景/背景：[畫面場景或背景]
核心元素：[必須出現的視覺元素]
構圖：[畫面主體位置]，[文字位置]，[資訊層級要求]
畫面文字白名單：[逐字列出允許出現的主標題、副標題、輔助文案、CTA、必要告知]
禁止文字：[列出原型圖中不得進入畫面的內部文字；沒有則填“除白名單外不得出現任何文字”]
風格要求：[貼入已確認的固定統一視覺風格描述]
尺寸：豎版電商詳情頁，寬 750px
```

限制要求：

- 畫面中只允許出現已確認的「畫面文字白名單」，不得因某段文字出現在原型圖、HTML 或規劃稿中就自動帶入。
- 資訊任務、視覺建議、分析結論、提示詞欄位名、風險提醒、假設、內部備註、自檢文字不得出現在畫面。
- 不要新增任何未經確認的標題、標籤、小字、證明、參數、功效解釋或 CTA。
- 不要打亂原型圖的資訊層級。
- 不要修改產品包裝。
- 不要出現錯誤文字。
- 不要直接生成圖片。
- 每一屏提示詞都要服務對應屏的資訊任務、視覺表達類型和核心視覺元素。

階段收尾話術：

```text
以上是逐屏 AI 生圖提示詞。你可以確認全部提示詞，也可以指定某一屏修改主體、背景、核心元素、構圖或風格要求。確認後，我會使用 ChatGPT Image 2 從第 1 屏開始逐屏生成。
```

## 階段 5：逐屏生成設計

每次只生成一屏。不要一次性生成全部屏。

生成當前屏圖片時，必須呼叫 `ChatGPT Image 2`。該模型更適合電商詳情頁裡的中文標題、賣點和長圖文案呈現。

如果當前環境無法呼叫 `ChatGPT Image 2`，不要改用其他生圖模型替代；應先說明限制，並輸出可複製給 `ChatGPT Image 2` 的單屏作圖提示詞。

單屏生成前，先明確：

```text
當前生成：第 X 屏
使用的 AI 生圖提示詞：
資訊任務：
沿用視覺風格：
核心視覺元素：
畫面文字白名單：
禁止上畫面文字：
需要避開的風險點：
```

單屏作圖要求：

- 寬度按使用者要求，預設 750px。
- 完整呈現當前屏文字白名單，不漏字、不改字、不壓縮成摘要。
- 除白名單外，不得生成任何額外文字。
- 畫面必須服務當前屏的資訊任務。
- 產品、賣點、使用場景或關鍵視覺元素要服務轉換。
- CTA 屏必須有清晰可辨識的 CTA 區域，並保持已確認的 CTA 文案。
- 不使用絕對化、醫療化、功效承諾或無法證明的資料表達。
- 不讓文字被產品、裝飾、邊緣或背景干擾。
- 嚴格按照已確認的當前屏 AI 生圖提示詞執行。
- 生成後先自行比對文字白名單與禁止文字；發現多字、漏字、錯字、內部文字或 CTA 遺漏時，立即重做當前屏，不把問題稿交給使用者確認。
- 通過單屏自檢後再等待使用者確認。

單屏生成後的固定話術：

```text
這是第 X 屏。你可以回覆“確認”，我再繼續下一屏；也可以告訴我需要修改的地方，例如產品更大、文字更突出、風格更清爽、場景更生活化、色彩更貼近包裝。
```

## 階段 6：當前屏修改確認

使用者提出修改意見時，只修改當前屏。

修改規則：

- 保留使用者已經確認的部分。
- 不影響其他屏。
- 修改後重新檢查文字白名單、禁止文字與 CTA，不得在修圖時帶入內部溝通內容。
- 如果使用者要求會導致文字裁切、合規風險或視覺衝突，先指出問題，再給替代方案。
- 修改完成後繼續等待確認。

修改後的話術：

```text
已按你的意見調整當前屏。你可以繼續提出修改，或回覆“確認”進入下一屏。
```

## 階段 7：進入下一屏

使用者確認當前屏後，繼續下一屏。

進入下一屏的話術：

```text
當前屏已確認。我會沿用已經確認的整體視覺風格，繼續生成第 X 屏；這一屏會根據它的資訊任務調整畫面重點，避免和上一屏構圖完全重複。
```

## 階段 8：長圖拼接

當所有單屏都確認後，再執行長圖拼接。

長圖要求：

- 順序不變。
- 寬度統一 750px，除非使用者另有要求。
- 不裁切文字和產品。
- 不壓縮導致文字模糊。
- 屏與屏之間銜接自然。
- 不漏屏、不錯序、不重複。

拼接前確認話術：

```text
所有單屏已經確認。我接下來會按確認順序縱向拼接成長圖，預設寬度 750px，並檢查漏屏、錯序、重複、文字裁切和產品裁切問題。
```

## 階段 9：最終品檢

長圖完成後必須做最終檢查，並輸出檢查結果。

品檢清單：

```text
長圖品檢結果：
- 螢幕順序：
- 是否漏屏：
- 是否重複：
- 寬度是否統一：
- 文字是否完整清晰：
- 是否只出現文字白名單：
- 是否混入內部溝通文字：
- CTA 是否按首屏、中段、末屏正確分布：
- CTA 文案與當前屏是否一致：
- 各屏文案是否服務銷售轉換：
- 除必要說明外，文案是否具備痛點/癢點/爽點/情境/共感/動作/利益：
- 產品是否被裁切：
- 屏與屏銜接：
- 合規風險複查：
```

如果發現問題，先列出問題，再修正，不要直接說完成。
